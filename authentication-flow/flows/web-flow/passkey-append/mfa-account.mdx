Users logging in via desktop and mobile browsers, follows a standard login flow for MFA user with passkey append experience.

<Steps>
    <Step title="Login when user has no passkeys and activated MFA">
        - User enters their identifier (email/username) on the login page.
        - Since no passkey is found, user is prompted to log in using their traditional password.
        - After successful password authentication, the user is prompted to verify their account via a code sent through email or SMS.
        - After successful verification, the user is offered the option to create a passkey for future logins.
        - User chooses to set up a passkey using fingerprint, face, or PIN for secure sign-in.
        - After successful passkey creation, the user is signed in and redirected to the app dashboard.
        </Step>

</Steps>
