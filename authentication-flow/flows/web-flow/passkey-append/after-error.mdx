## Passkey Creation after Error

Users log in on desktops & mobile, experience an error during passkey login, need to use conventional login methods and can then append a new passkey.

<Steps>
    <Step title="Login when user has no passkeys">
        - User enters their identifier (email/username) on the login page.
    </Step>
    <Step title="Fallback to traditional login method">
        - System detects no passkey created for the respective account, system fallbacks to traditional login method (username + password).
    </Step>
    <Step title="Passkey append process initiates">
        - User is offered an option to create a passkey for future logins.
    </Step>
    <Step title="Passkey append process abruptly cancels or an error occurs">
        - There is an abrupt error faced during passkey creation process, the user is not negatively notified for the error. Instead, an informing screen appears where the user is again asked to create a passkey.
    </Step>
    <Step title="OS-native modal overlay">
        - User continue to set up a passkey using fingerprint, face, or PIN for secure sign-in.
    </Step>
    <Step title="Passkey creation successful">
        - User successfully creates passkey and is now linked with the user account for the respective web-app.
    </Step>
    <Step title="Redirection to app dashboard">
        - User is redirected to the app dashboard.
    </Step>

</Steps>
