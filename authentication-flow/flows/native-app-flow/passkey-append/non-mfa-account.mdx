## Passkey Creation For Non-MFA User Accounts

Users logging in native app, follows a standard login flow for non-MFA user with passkey append experience.

<Steps>
     <Step title="Login when user has no passkeys">
        - User enters their identifier (email/username) on the login page.
        - Since no passkey is found, user is prompted to log in using their traditional password.
        - After successful password authentication, the user is offered the option to create a passkey for future logins.
    </Step>
    <Step title="Creating a passkey after login">
        - User chooses to set up a passkey using fingerprint, face, or PIN for secure sign-in.
        - After creating the passkey, the user is prompted to verify their account via a code sent through email or SMS.
        - After successful verification, the user is signed in, asked to setup PIN and redirected to the app
        dashboard.
    </Step>
    <Step title="Skipping the passkey creation">
        - User skips the passkey setup process.
        - User is signed in with password and is asked to verify their identity via code verification.
        - User is asked to setup PIN and is directed to the app dashboard as usual. Users then have an option to enable local biometric login.
    </Step>

</Steps>
