## Token validation (JWT)

Since there is no official Corbado C#/.NET SDK, you need to validate session tokens (JWTs) manually in your backend. This ensures that only valid, untampered tokens are accepted for authentication.

To validate a Corbado session token (JWT) in C#, use a JWT library such as [System.IdentityModel.Tokens.Jwt](https://www.nuget.org/packages/System.IdentityModel.Tokens.Jwt/). You need to fetch the JWKS (public keys) from Corbado and verify the token's signature and claims.

- Verifies the JWT signature using Corbado's public keys (JWKS).
- Checks the token's expiration and validity.
- Returns the decoded claims if the token is valid, or throws an error otherwise.

### How to use

```csharp
using System.IdentityModel.Tokens.Jwt;
using Microsoft.IdentityModel.Tokens;
using System.Net.Http;
using System.Text.Json;

// Fetch the session token from a cookie or Authorization header.
// See session token handling for details: https://docs.corbado.com/corbado-complete/sessions/session-token-handling
string sessionToken = "...";

// Fetch JWKS from Corbado (example using HttpClient)
string jwksUrl = "https://<your-project-id>.frontendapi.corbado.io/.well-known/jwks.json";
HttpClient httpClient = new HttpClient();
string jwksJson = await httpClient.GetStringAsync(jwksUrl);
// Parse JWKS and extract the public key (see System.IdentityModel.Tokens.Jwt docs)
// ...

var tokenHandler = new JwtSecurityTokenHandler();
var validationParameters = new TokenValidationParameters
{
    ValidateIssuer = true,
    ValidIssuer = "https://<your-project-id>.frontendapi.corbado.io",
    ValidateAudience = false,
    ValidateLifetime = true,
    RequireExpirationTime = true,
    ValidateIssuerSigningKey = true,
    IssuerSigningKey = new RsaSecurityKey(publicKeyFromJwks), // Replace with actual key extraction
    // ... other parameters ...
};

try
{
    var principal = tokenHandler.ValidateToken(sessionToken, validationParameters, out var validatedToken);
    // Token is valid, use claims from principal
}
catch (Exception ex)
{
    // Token is invalid
}
```

### Why is this important?
- Ensures only valid, untampered tokens are accepted.
- Protects your backend endpoints from unauthorized access.
- Handles key rotation and validation logic for you.

See [session token handling](/corbado-complete/sessions/session-token-handling) for more details.
