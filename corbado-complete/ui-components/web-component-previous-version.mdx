## Overview

The following page shows how to set up the pre-built Corbado web component. Using the web component saves you a lot of engineering time, comes with many of out-of-the-box features and can be easily integrated with existing authentication solutions.

<Check>
Before you start integrating, please follow [Getting started](/corbado-complete/overview/getting-started) to setup your project at the developer panel.

If you use React or VanillaJS as your tech stack, we highly recommend to use the new [React components](/corbado-complete/frontend-integration/react) or [VanillaJS components](/corbado-complete/frontend-integration/vanilla-js).

</Check>

### 1. Embed the web component into your site

You can install the Corbado web component via npm or add the JavaScript library via CDN.&#x20;

<Card>

Install the [web component](https://www.npmjs.com/package/@corbado/webcomponent) via NPM:

```bash
npm install @corbado/webcomponent
```

Next, include the package in the module where you plan to utilize the web component, and import the styling file accordingly:

```javascript
import "@corbado/webcomponent/pkg/auth_cui.css";
import "@corbado/webcomponent";
```

</Card>
<Card>
Use the web component via CDN:

To add the Corbado JavaScript library to your website, simply insert the following code on the same page where you want to add the web component.

<Warning>
Must be put in `<body>`part,  **not** in`<head>`!
</Warning>

```html
<script src="https://<Project ID>.frontendapi.cloud.corbado.io/auth.js"></script>
```

</Card>

Embed the Corbado web component by adding `<corbado-auth/>` to your login page:

```html
<!-- Your website content -->

<corbado-auth project-id="<Project ID>" conditional="yes">
  <input
    name="username"
    id="corbado-username"
    required
    autocomplete="webauthn"
  />
</corbado-auth>

<!-- Your website content -->
```

### 2. Add additional HTML attributes (optional)

The following attributes for `<corbado-auth/>` exist:

| Attribute               | Explanation                                                                                                                                                                                                                                                            | Default value                                    |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| project-id              | This should be the `project ID` of your project.                                                                                                                                                                                                                       | -                                                |
| company-image-url       | You can provide a URL for the company logo which will be displayed in the top of the web component. For best performance, the logo should have a maximum height of 80px.                                                                                               | -                                                |
| conditional             | If you want to use Conditional UI, you need to set this attribute to "yes" and add the inner `<input/>` element.                                                                                                                                                       | no                                               |
| page                    | You can decide if "login" or "register" page should be displayed at first, when somebody sees the web component.                                                                                                                                                       | login                                            |
| register-title          | Change the title of the sign-up screen of the web component.                                                                                                                                                                                                           | null                                             |
| register-btn            | Change the button text of the sign-up button of the web component.                                                                                                                                                                                                     | null                                             |
| login-title             | Change the title of the login screen of the web component.                                                                                                                                                                                                             | null                                             |
| login-btn               | Change the button text of the login button of the web component.                                                                                                                                                                                                       | null                                             |
| sub-title               | Change the sub tile of the login screen of the web component (it can also be empty).                                                                                                                                                                                   | Welcome back! Please enter your details          |
| endpoint                | This should be your [Frontend API](/corbado-complete/overview/corbado-api-structure/frontend-api).                                                                                                                                                                           | `https://<project ID>.frontendapi.cloud.corbado.io`    |
| username                | Pre-fill the username (email address) input field with a value of your choice. If there is no username HTML attribute set or it is empty, LocalStorage is searched for a key `username` that is then taken instead. If that doesn't exist, the pre-fill will be empty. | -                                                |
| user-full-name          | Pre-fill the userFullName input field with a value of your choice.                                                                                                                                                                                                     | -                                                |
| passkey-append-text     | Change the text of the passkey-append-screen                                                                                                                                                                                                                           | Login event faster with Touch ID, Face ID or PIN |
| disable-input-animation | Remove the animation of the labels for input fields                                                                                                                                                                                                                    | no                                               |
| email-button-title | Change the title of the email magic link button fields                                                                                                                                                                                                                    | null                                               |
| resend-email-text | Change the text of resend email fields                                                                                                                                                                                                                    | null                                               |

<Info>
  The inner `<input />` element is needed to make [Conditional
  UI](https://www.corbado.com/blog/webauthn-conditional-ui-passkeys-autofill)
  available as some browsers currently do not support Conditional UI in web
  components.
</Info>

If you want to use [Conditional UI](https://www.corbado.com/blog/webauthn-conditional-ui-passkeys-autofill) (what we recommend for the best UX), just copy and paste the inner `<input/>` element's attributes like shown above. If not, you don't need to insert the inner `<input/>` element at all.

<Info>
  Remember that the `<corbado-auth/>` web component varies in height depending
  on the number of textfields / buttons that are currently visible.
</Info>

### 4. Add session management (optional)

#### 4.1 Session refresh

If you want to use Corbado's [session management](/corbado-complete/sessions/overview), it is mandatory to include the JavaScript logic below on **all** of your pages. This ensures that [session refresh](/corbado-complete/sessions/other/refresh) functionality works seamlessly.

#### 4.2 Session logout

Corbado provides functionality to logout the current user. The logout work as follows:&#x20;

1. Corbado sends an [AJAX](https://www.w3schools.com/js/js_ajax_intro.asp) call to your [Frontend API](/corbado-complete/overview/corbado-api-structure/frontend-api).
2. The current session is revoked using your **long-term** session (represented as session ID with database entry).
3. In the response of the [AJAX](https://www.w3schools.com/js/js_ajax_intro.asp) call, Corbado sends a delete cookie.
4. The **short-term** session cookie (represented as JWT) is deleted via JavaScript.

<CodeGroup>
```javascript React
import { useEffect } from 'react';
import Corbado from '@corbado/webcomponent';

function App() {
const session = new Corbado.Session('<Project ID>');

    useEffect(() => {
        // Register a callback for session refresh
        // in order to receive an authentication event
        session.refresh(user => {
            // Here, you can define what happens when the session is initialized
            // if user is null, then the user is not logged in
        });
    }, [session]);

    const handleLogout = () => {
        session.logout()
            .then(() => {
                // Here, you can define what happens after the user is logged out
            })
            .catch(err => {
                console.error(err);
            });
    }

    return (
        <div>
            {/* Render your UI here */}
            <button onClick={handleLogout}>Logout</button>
        </div>
    );

}

export default App;
```

```html Vue.js
<template>
  <div>
    <!-- Render your UI here -->
    <button @click="handleLogout">Logout</button>
  </div>
</template>

<script>
  import Corbado from "@corbado/webcomponent";

  export default {
    name: "App",
    setup() {
      return {
        session: new Corbado.Session("<Project ID>"),

      // Register a callback for session refresh

      this.session.refresh((user) => {
        // Here, you can define what happens when the session is initialized
        // if user is null, then the user is not logged in
      });
    },
    methods: {
      handleLogout() {
        this.session
          .logout()
          .then(() => {
            // Here, you can define what happens after the user is logged out
          })
          .catch((err) => {
            console.error(err);
          });
      },
    },
  };
</script>
```

```html Vanilla HTML / JS
<!DOCTYPE html>
<html>
  <head>
    <title>Corbado</title>
  </head>
  <body>
    <!-- Note, that the following scripts HAVE to be inside the body tag -->

      const session = new Corbado.Session("<Project ID>");

      // Register a callback for session refresh

        // if user is null, then the user is not logged in
      });

      // Get logout button
      const logoutButton = document.getElementById("logoutButton");

      // Add event listener to logout button
      logoutButton.addEventListener("click", function () {
        session
          .logout()
          .then(() => {
            window.location.replace("/");
          })
          .catch((err) => {
            console.error(err);
          });
      });
    </script>
  </body>
</html>
```

### 5. Define Application URL

Please provide the URL where the web component runs (incl. protocol, port & path) as [Application URL](/corbado-complete/helpful-guides/url-overview#.-application-url) in ["Settings" - "General" - "URLs"](https://app.corbado.com/app/settings/general/urls). The [Application URL](/corbado-complete/helpful-guides/url-overview#2-application-url) is used, among others, to correctly direct users to the web component again, when they have clicked on an email magic link.

Often, it is on the same domain like the [authorized origin](/corbado-complete/helpful-guides/url-overview#1-authorized-origin).

![Application URL in Developer Panel](/images/frontend-integration/ui-components/web-component/application-url-setting.webp)

### 6. Define Redirect URL

Once a user has authenticated successfully (either with passkey, email magic link or password), the user redirected to the [Redirect URL](/corbado-complete/helpful-guides/url-overview#3-redirect-url)in the browser.

Provide the [Redirect URL](/corbado-complete/helpful-guides/url-overview#3-redirect-url) (incl. protocol, port and path) in ["Settings" - "General" - "URLs"](https://app.corbado.com/app/settings/general/urls), e.g.:

- `https://www.my-application.com/redirectUrl` for a live application
- `http://localhost:8080/redirectUrl` for localhost

![Redirect URL in Developer Panel](/images/frontend-integration/ui-components/web-component/redirect-url-setting.webp)

### 7. Verify Relying Party ID

By default the WebAuthn [Relying Party ID](/corbado-complete/helpful-guides/url-overview#7-relying-party-id) will be set to `<project ID>.preview.corbado.io` for previews and `localhost` for web component integration. This is where the passkeys are bound to. You can only use passkeys in a web component that runs in a matching domain of the [Relying Party ID](/corbado-complete/helpful-guides/url-overview#7-relying-party-id). Matching domains means that it's either the same or a subdomain of the [Relying Party ID](/corbado-complete/helpful-guides/url-overview#7-relying-party-id).&#x20;

Provide the [Relying Party ID](/corbado-complete/helpful-guides/url-overview#7-relying-party-id) (without protocol, port and path) in ["Settings" - "General" - "URLs"](https://app.corbado.com/app/settings/general/urls), e.g.:

- `my-application.com` for a live application
- `localhost` for localhost

<Info>
If you develop locally on `http://localhost:<port>`, please set the [Relying Party ID](/corbado-complete/helpful-guides/url-overview#7-relying-party-id) to `localhost.`

We highly recommend to take at our [detailed blog post about the Relying Party ID](https://www.corbado.com/blog/webauthn-relying-party-id-rpid-passkeys).

</Info>

![Relying Party ID in Developer Panel](/images/overview/signup/passkey-configuration/relying-party-id.webp)
