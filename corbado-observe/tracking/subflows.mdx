Subflows are the concrete building blocks inside a [flow](/corbado-observe/tracking/flows). They represent a specific authentication step, for example password login, passkey enrollment, or email OTP verification.

Every tracking event type maps to exactly one subflow. This gives you clear and consistent funnel nodes for detailed analysis.

<Info>
  Install and set up the **Corbado Observe SDK**, including your first tracked event, in [Getting started](/corbado-observe/overview/getting-started). This page focuses only on subflow-to-event mapping.
</Info>
