Source 2:

A decision is a tracked choice point — either by the **user** or the **system** — within a flow. Decisions are defined by the integrating developer, not auto-discovered.

| Decision | ID | Type | Relevant Flows |
| --- | --- | --- | --- |
| Pre-identifier | d0 | user | login, signup |
| Post-identifier | d1 | user | login |
| Enrollment-allowed | d2 | system | login, signup |
| Prefilling-accepted | d3 | system/user | login |

**Decision anatomy:**

- Has **available options** (can change within a single flow)
- Has **outcomes** (0, 1, or more per flow)
- User can make the same decision **multiple times** in a flow (e.g., retry after error)
- System can make decisions automatically (e.g., auto-switch after failed passkey)

**Decision lifecycle:**

- **Starts** when options become visible
- **Ends** when an outcome is selected (subflow state changes from offered → startable, or startable → started)

**Visualization:**

- End-state funnel (last decision outcome)
- DGF / Decision Graph Flow (all decision outcomes)
- Decision input/outcome analysis panels
- Observe funnel filters
