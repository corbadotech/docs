## Overview

Webhooks are user-defined HTTP callbacks that trigger automated messages or updates to external systems based on specific events or conditions in a web service. They enable real-time communication between applications by sending data payloads to a configured URL when an event occurs, facilitating seamless integration and automation.

## 1. Corbado webhooks actions

Corbado offers the following webhooks actions:

| Action                                                                                          | Description                                                                                                         |
| ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| [authMethods](/helpful-guides/webhooks/action-authmethods)       | This action returns whether the user already exists in the user database and which authentication methods are already in place. |
| [passwordVerify](/helpful-guides/webhooks/action-passwordverify) | This action verifies if the provided username and password combination of the existing authentication is valid.     |
| [events](/helpful-guides/webhooks/action-events) | This action sends out messages for specific user-related events such as creation, deletion, and updates.|



## 2. Webhooks requests

Webhook requests from Corbado sent to your backend follow this payload structure:

```json
{
  "id": "who-1234567890", // Unique ID per webhook request
  "projectID": "pro-1234567890", // Your project ID from the developer panel
  "action": "authMethods", // The specific action of the webhook request, see table above
  "data": {
    // Data object dependent on action
  }
}
```

The action is sent as a header (`X-Corbado-Action`) as well. This makes it easier for you to deserialize the payload into an action-dependent entity / model by taking the action from the header instead of from the payload itself."

## 3. Webhooks responses

Webhook responses from your backend sent to Corbado follow this payload structure:

```json
{
  "responseID": "d5a80602-a771-4532-8cc8-6d4a9003d92a", // A response ID you can set in your backend for debugging
  "data": {
    // Data object dependent on action
  }
}
```


## 4. Backend implementation

Plus, they come with entities / models for request and response (dependent on action) and ease authentication handling for you. We highly encourage you to use them.
