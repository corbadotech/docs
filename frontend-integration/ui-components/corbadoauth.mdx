## Introduction

![CorbadoAuth Component](/images/frontend-integration/ui-components/corbado-auth/corbadoauth-component.webp)

\<CorbadoAuth/> combines the \<SignUp/> and the \<Login/> components and wires them together. Use this UI component if you don't want to take care of authentication-related navigation inside your application.&#x20;

<CodeGroup>

```tsx React
import { CorbadoAuth } from "@corbado/react";

const AuthPage = () => {
  const navigate = useNavigate();

  const onLoggedIn = () => {
    navigate("/");
  };

  return <CorbadoAuth onLoggedIn={onLoggedIn} />;
};
```

```html VanillaJS
<script>
  const authElement = document.getElementById("corbado-auth");

  Corbado.mountAuthUI(authElement, {
    onLoggedIn: () => {
      window.location.href = "/";
    },
  });
</script>

<div id="corbado-auth"></div>
```

</CodeGroup>

### API reference

| Name       | Type              | Description                                                                                                                                                                                                               |
| ---------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| onLoggedIn | callback function | This callback function is called whenever a user has been authenticated (i.e., after every login, but also after a signUp). You can use it to navigate the user to a specific page inside your application (e.g., /home). |

**Note:** This component must be wrapped inside a \<CorbadoProvider/> component. Most configuration settings will be applied there so make sure to check out the available settings [here](./corbadoprovider).
