## Introduction

`CorbadoAuth` serves as a comprehensive solution for user authentication, combining both account creation and login functionalities into a single, streamlined interface.

## Example

<CodeGroup>

```tsx React
import { CorbadoAuth } from "@corbado/react";

const AuthPage = () => {
  const onLoggedIn = () => {
    //post login actions can be performed here.
  };

  return <CorbadoAuth onLoggedIn={onLoggedIn} />;
};
```

```html VanillaJS
<script>
  const authElement = document.getElementById("corbado-auth"); //Element where you want to render CorbadoAuth UI

  Corbado.mountAuthUI(authElement, {
    onLoggedIn: () => {
      //post login actions can be performed here.
    },
  });
</script>

<div id="corbado-auth"></div>
```

</CodeGroup>

## API reference

| Name                   | Type              | Description                                                                                                                                                                                                               |
| ---------------------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| onLoggedIn             | callback function | This callback function is called whenever a user has been authenticated (i.e., after every login, but also after a signUp). You can use it to navigate the user to a specific page inside your application (e.g., /home). |
| handleNavigationEvents | boolean           | Can be use to enable and disable hash-based urls. Defaults to `true`. Hash-Based URL navigation is used manage the signup flow state but can be disabled if it interferes with the user's application's functionalities.  |
